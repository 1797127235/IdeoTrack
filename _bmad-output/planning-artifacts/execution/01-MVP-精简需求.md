## 2. 项目概述

IdeoTrack（思政打卡 App）是一款面向高校大学生的移动端学习打卡工具，旨在替代当前依赖微信群接龙、班长手工统计或 Excel 汇总的低效思政学习管理模式。通过将打卡行为从单纯的「任务」转化为学生可感知成就感与集体荣誉的「每日积累」，产品帮助学生便捷完成思政学习任务，同时为辅导员和管理员提供实时、可视化的数据管理与汇报能力。

MVP 阶段的核心目标是实现 **三条主干链路 100% 可跑通**：学生完成每日打卡、辅导员实时查看班级数据并处理异常、管理员快速汇总全校数据并生成报告。在此基础上，通过积分、等级、连续打卡日历和班级排行榜等轻量激励机制，提升学生完成率与体验。

---

## 3. 核心目标用户

| 角色 | 一句话描述 |
|------|-----------|
| **学生（大学生）** | 在手机上快速完成学校布置的每日思政学习任务，同时看到自己的进步和班级表现。 |
| **辅导员** | 快速掌握所带班级（可跨多个班）的打卡完成情况，及时发现异常并督促完成。 |
| **管理员** | 高效汇总全校思政学习数据，生成可向上级主管部门或校级领导汇报的工作报告。 |

## 4. MVP 功能清单

> **优先级说明：**  
> - **P0** — 主干链路必备功能，必须实现。  
> - **P1** — 重要增强功能，在主干稳定后实现，若工期紧张可延后。

### 4.1 学生端

| 模块 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 账号与登录 | 学号/工号密码登录 | P0 | 学生使用学号+密码登录；连续 5 次输入错误锁定 15 分钟；首次登录强制修改初始密码。 |
| 账号与登录 | 角色权限控制 | P0 | 学生角色仅能访问学生端功能（任务、打卡、个人中心、排行榜）。 |
| 首页与每日名言 | 每日名言展示 | P0 | 首页展示每日名言/励志语录；同一自然日内保持一致；支持左右滑动查看近 7 日历史。 |
| 任务管理 | 任务列表展示 | P0 | 首页展示当日及历史任务列表；按截止时间倒序排列；每个任务卡片显示标题、截止时间和完成状态。 |
| 任务管理 | 任务详情查看 | P0 | 展示任务标题、内容、发布时间和截止时间；提供「去打卡」入口；已完成的任务展示提交的心得。 |
| 打卡流程 | 定位签到 | P0 | 点击签到后系统记录当前经纬度、地址描述和签到时间；拒绝定位时提示开启权限；同一任务以最后一次签到为准。 |
| 打卡流程 | 心得提交 | P0 | 最少 10 字，最多 500 字；纯文本输入；提交后进入 AI 初审队列；在辅导员复核前可修改 1 次。 |
| 打卡流程 | 打卡完成反馈 | P0 | AI 初审通过：显示「打卡成功」，更新积分、连续打卡天数和打卡日历；未通过：显示「需要复核」，可补充内容或等待辅导员复核。 |
| 心得审核 | 学生查看复核结果 | P0 | 在个人打卡记录中查看每条心得的审核状态（已通过/待复核/未通过/要求修改）；「要求修改」时可重新提交，「未通过」时显示原因说明。 |
| 激励机制 | 积分计算 | P0 | 每次有效打卡获得 10 基础积分；连续打卡第 2 天起每天额外 +2 分，上限额外 +10 分（即连续 6 天及以上每天最高 20 分）；断签后连续天数和加成从头计算，等级保留不降级。 |
| 激励机制 | 等级晋升 | P0 | 青铜（0–99 分）、白银（100–499 分）、黄金（500–999 分）、铂金（1000 分以上）；达到对应阈值自动提升；在个人中心首页展示。 |
| 激励机制 | 打卡日历展示 | P0 | 月视图展示每日打卡状态（已打卡绿色/未打卡灰色）；点击日期可查看当日心得内容；V1 不支持补卡。 |
| 激励机制 | 勋章授予 | P1 | 连续打卡 7 天获得「坚持一周」勋章；连续打卡 30 天获得「坚持一月」勋章；通过审核后立即授予，在打卡成功页弹出提示。 |
| 班级排行榜 | 班级打卡率排名 | P1 | 默认展示今日班级打卡率排名；支持切换时间范围：今日、本周、本月；只展示班级名称和班级打卡率，不展示个人数据；排名前 3 的班级有特殊标识。 |
| 班级排行榜 | 排行榜数据刷新 | P1 | 系统定时计算并刷新班级排行榜数据，允许最多 5 分钟延迟，不依赖学生手动下拉。 |

### 4.2 辅导员端

| 模块 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 账号与登录 | 学号/工号密码登录 | P0 | 辅导员使用工号+密码登录；首次登录强制修改初始密码。 |
| 账号与登录 | 角色权限控制 | P0 | 辅导员角色仅能访问所带班级的数据看板和复核功能，不能跨班级访问。 |
| 数据看板 | 班级数据概览 | P0 | 首页展示所带每个班级的今日打卡率、应打卡人数、已打卡人数；支持按班级下钻查看详细名单；数据允许最多 5 分钟延迟。 |
| 数据看板 | 未打卡学生名单 | P0 | 展示学生姓名、学号、连续未打卡天数；连续 3 天及以上未打卡的学生标红显示；支持按班级、日期筛选。 |
| 数据看板 | 辅导员数据导出 | P0 | 支持导出 Excel 格式；导出范围可选指定班级和日期范围；导出内容包含学生姓名、学号、打卡状态、打卡时间、心得内容、审核状态。 |
| 心得审核 | 辅导员人工复核 | P0 | 在「待复核」列表查看本班 AI 初审未通过的心得；可选择「通过」「不通过」或「要求修改」；通过后打卡状态变为「已完成」并发放积分；要求修改后学生可重新提交。 |
| 数据看板 | 一键提醒 | P1 | 选择未打卡学生后，批量发送 App 内推送消息；同一学生每天最多收到 1 次提醒，发送时间限制在 8:00–22:00；发送记录在看板中保存。 |

### 4.3 管理员端

| 模块 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 账号与登录 | 学号/工号密码登录 | P0 | 管理员使用工号+密码登录；首次登录强制修改初始密码。 |
| 账号与登录 | 角色权限控制 | P0 | 管理员角色可访问任务发布、组织管理、全校报告等后台功能。 |
| 账号与登录 | 用户管理 | P0 | 支持单个添加和 Excel 模板批量导入用户；初始密码默认为学号/工号后 6 位；支持分配角色和所属班级/学院；支持启用/禁用账号；V1 不支持用户自助注册。 |
| 首页与每日名言 | 名言库配置 | P0 | 支持文本内容录入，单条不超过 200 字；可添加、编辑、删除名言，并设置启用状态；启用数量不足时系统循环使用；根据启用状态按日轮换展示每日名言。 |
| 任务管理 | 发布任务 | P0 | 创建任务必须包含标题、内容、发布时间和截止时间；可指定发布范围（全校/某个学院/某个班级）；截止前可编辑或下架；发布人可查看任务完成率统计。 |
| 组织结构 | 组织结构管理 | P0 | 支持创建、编辑、删除学院；支持在学院下创建、编辑、删除班级；班级必须属于一个学院；删除班级前检查是否仍有学生并提示。 |
| 报告与统计 | 多维度数据统计 | P0 | 按全校/学院/班级/时间段查看统计数据；统计维度包括打卡率、未打卡人数、心得提交数、AI 初审通过率、人工复核数；数据以图表（折线图、柱状图）和表格形式展示；异常数据点（较过去 7 天平均值下降 ≥ 20 个百分点）标注「建议核查」提示。 |
| 报告与统计 | 报告导出 | P0 | 支持 Excel 和 PDF 两种格式；PDF 报告必须包含封面、摘要、趋势图、排名图、明细表；Excel 包含原始数据；导出范围可选全校、指定学院、指定班级、指定时间范围。 |

### 4.4 系统级功能（通知与提醒）

| 模块 | 功能点 | 优先级 | 说明 |
|------|--------|--------|------|
| 通知与提醒 | 打卡提醒 | P1 | 每日固定时间（默认 20:00）向当日未打卡学生发送 App 内推送；学生可在设置中关闭提醒。 |
| 通知与提醒 | 复核结果通知 | P1 | 辅导员复核后，学生收到「通过」「不通过」或「要求修改」的通知；通知可跳转至对应打卡记录详情。 |
| 通知与提醒 | 系统通知 | P1 | 通知中心展示历史通知列表；类型包括任务发布、任务变更、账号状态变更等；未读通知显示红点标记。 |

## 5. 技术约束与假设

| 类别 | 约束 / 假设 |
|------|------------|
| **平台** | 优先支持 Android 端；如使用跨平台框架（Flutter / React Native）则同时覆盖 iOS，但以 Android 为主要验证端。支持主流手机屏幕尺寸（5.5–6.7 英寸）。 |
| **部署** | 单一学校部署，无需多租户隔离。V1 不要求 24/7 高可用，但核心打卡服务在高峰时段（20:00–22:00）应稳定可用。 |
| **网络** | 假设基本网络可用，V1 不支持复杂离线同步。 |
| **数据安全** | 用户密码必须加密存储（建议 bcrypt）；学生位置信息和心得内容属于敏感数据，传输必须使用 HTTPS；辅导员只能查看所带班级数据，不能跨班级访问；管理员可查看全校汇总数据，但不应随意查看个人心得详情（除非用于复核审计）。 |
| **AI 初审** | 规则在 V1 阶段由系统预设，无需管理员配置。检测规则：字数 < 10 判定为敷衍；与任务原文 Jaccard 相似度 ≥ 70% 或命中 50 条常见套话模板判定为敷衍；命中预设敏感词库判定为需复核。AI 初审结果在 3 秒内返回（非高峰期）。 |
| **数据备份** | 每日备份核心数据（用户、打卡记录、任务）。 |
| **合规** | 在正式采集学生位置信息和心得内容前，需经学校相关部门（学工处、信息中心）审核确认数据用途、存储期限和访问范围，并制定数据删除/retention 策略。 |
| **内容** | V1 任务内容仅支持纯文本，不支持富文本、图文混排或视频嵌入。心得提交仅支持纯文本，不支持图片、语音、视频附件。 |

## 6. 验收标准

> 以下验收条件对应 PRD §6「Success Metrics」，为可测试的验证条件。

| 编号 | 验收条件 | 对应 PRD 指标 | 验证方式 |
|------|----------|---------------|----------|
| AC-1 | 演示数据集下，当日有效打卡学生数 ÷ 当日应打卡学生数 ≥ **80%**。 | SM-1 | 功能测试 / 数据核对 |
| AC-2 | 辅导员从登录到查看所带班级当日打卡率的点击次数 ≤ **2 次**。 | SM-2 | 任务测试 / 操作计时 |
| AC-3 | 正常网络和数据规模下，管理员导出 Excel/PDF 报告的成功率 ≥ **95%**，且无未处理异常。 | SM-3 | 功能测试 / 异常监控 |
| AC-4 | 学生从打开任务详情到完成打卡（查看任务 → 签到 → 写心得 → 提交）的步骤数 ≤ **4 步**。 | SM-4 | 任务测试 / 操作计数 |
| AC-5 | AI 成功完成初审的心得数 ÷ 总提交心得数 ≥ **99%**。 | SM-5 | 日志审计 / 数据核对 |

## 7. 明确排除项（V1 不做）

> 以下功能在 PRD §5.2「Out of Scope for MVP」中明确列为 V1 之后迭代方向，开发团队不得纳入 MVP 范围。

| 排除项 | 原因 | 后续规划 |
|--------|------|----------|
| 积分兑换奖品 / 商城运营闭环 | 引入库存、物流、审批等运营复杂度 | V2 视运营能力决定 |
| 地理位置围栏校验 | 增加定位权限策略和合规风险 | V2 如需考勤精准化再引入 |
| 虚拟定位 / 作弊检测 | 需要专门的风控策略和数据验证 | V2 如需严格考勤再引入 |
| 个人排行榜 / 同学互评 | 隐私保护和竞争压力风险 | V2 可试点，需学校同意 |
| 社交互动（评论、点赞、班级群聊等） | 内容审核成本大幅增加 | V2 作为可选模块 |
| 即时通讯 | 不属于核心打卡场景 | 一般不做 |
| 多学校 / SaaS 化 | 需要租户隔离和多组织权限体系 | V2 如需推广多校再引入 |
| 复杂离线同步 | V1 假设基本网络可用 | V2 网络不稳定场景优化 |
| 第三方登录 / 学校 SSO | 需要与学校信息中心对接 | V2 推广时对接 |
| 多媒体心得附件（图片、语音、视频） | 增加存储和审核成本 | V2 评估必要性 |
| 深度 AI 分析（情感分析、学习效果评估） | 超出 V1 管理报表需求 | V2 数据驱动场景 |
| 短信 / 邮件通知 | V1 通过 App 内推送已满足基本需求 | V2 如需强制触达再引入 |
| 用户自助注册 | 账号由管理员预录入或批量导入 | V1 通过管理员管理 |
| 忘记密码自助找回 | 通过管理员重置，不开放自助找回 | V1 通过管理员重置 |
| 任务内容富文本 / 图文混排 / 视频嵌入 | 增加内容审核和渲染复杂度 | V2 评估必要性 |
| 报告导出水印或审批流程 | 普通 PDF/Excel 导出即可满足 V1 需求 | V2 视学校合规要求决定 |
| 补卡功能 | 打卡日历仅展示已打卡/未打卡两种状态 | V2 视业务需求决定 |
| 一人多角色 | 一个用户只能有一个角色 | V2 视业务需求决定 |
