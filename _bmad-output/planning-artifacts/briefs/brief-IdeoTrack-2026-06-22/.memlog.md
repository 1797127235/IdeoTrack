- (decision) Core user stories identified: (1) Student submits daily check-in via mobile for ideological-education tasks; (2) Counselor views class check-in status to monitor safety and learning; (3) Admin aggregates school-wide data to generate education work reports.
- (decision) Check-in content: location-based sign-in plus student-written reflections/heartfelt notes. Platform: mobile app. Student submissions require review/approval.
- (assumption) Task publishing mechanism is currently undefined; will propose a default (admin/counselor publishes daily/weekly tasks) for user validation.
- (event) Stakes clarified: internship assignment, user wants to produce high-quality work. Gamification/incentive mechanisms parked for future discussion.
- (assumption) Proposed default: tasks are published by admin/counselor on a daily/weekly cycle; students see assigned tasks and complete check-ins. Awaiting user confirmation.
- (decision) User selected Coaching path for product-brief discovery: walk through assumptions section by section with pushback, then draft collaboratively.
- (decision) Target users: university students; counselors may manage multiple classes and need real-time data plus export; admin reports likely for superiors (detail TBD).
- (decision) Success criteria for internship assignment: complete features, visually appealing UI, user-friendly experience.
- (assumption) Problem framing assumed: current ideological-education check-in relies on manual collection (WeChat/Excel), causing incomplete data, delayed visibility for counselors, and heavy reporting workload for admins. Pending user validation.
- (event) Drafted sections 1-3 of brief.md: Problem & Opportunity, Target Users, Success Criteria. Awaiting user review before proceeding to core scenarios/features.
- (decision) Task publishing confirmed: admin/counselor publishes tasks; students complete check-ins against assigned tasks.
- (decision) Location check-in v1: student uploads location only; no geofencing or validation in MVP.
- (decision) Reflection content moderation: AI pre-review first; if flagged, escalates to counselor manual review. Review criteria: perfunctory content and sensitive words.
- (decision) Real-time data tolerance: a few minutes of delay is acceptable for counselor/admin dashboards.
- (event) Drafted sections 4-6: Core Scenarios, Core Feature Boundaries, and proposed Report Dimensions. Awaiting user confirmation on report dimensions and overall direction.
- (decision) Report dimensions confirmed by user: overall check-in rate, missing/exception list, rankings, trends, reflection submission/review stats, Excel/PDF export. No additions.
- (event) Applied structural and prose editorial reviews. Finalized brief.md and addendum.md with status ready-for-review.
- (change) Expanded V1 scope based on user input: added daily quote, personal check-in calendar/streak medals, basic points/levels, and class-level leaderboard. Prize redemption and advanced social features remain V2/addendum.
