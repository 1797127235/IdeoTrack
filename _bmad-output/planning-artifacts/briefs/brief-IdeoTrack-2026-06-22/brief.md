# 思政打卡 App 产品简介

> 本文件由 `bmad-product-brief` 创建，已完成教练式路径探索（Coaching-path Discovery）。

## 1. 问题与机会

当前高校的思政学习任务打卡常依赖微信群接龙、班长手工统计或 Excel 汇总，存在明显低效：

- 学生容易漏打，补交流程麻烦，也不清楚自己的完成进度。
- 辅导员跨班级手动收集数据，难以实时掌握学生状态。
- 管理员生成全校思政教育工作报告耗时费力，数据准确性依赖人工核对。

本 App 通过一个移动端打卡平台，让**学生便捷完成每日思政学习任务**，让**辅导员实时查看并导出班级数据**，让**管理员快速汇总全校数据并生成报告**，从而提升思政教育工作的效率与可追溯性。

同时，通过积分、连续打卡、日历可视化、班级排行榜等轻量激励机制，把打卡从单纯的「任务」转化为学生能感受到成就感和集体荣誉的「积累」。

## 2. 目标用户

| 用户 | 核心诉求 | 关键行为 |
|------|---------|---------|
| **学生（大学生）** | 在手机上快速完成每日思政学习任务，并获得成就感 | 查看任务 → 定位签到 → 提交心得 → 查看积分/等级/日历/班级排名 |
| **辅导员** | 掌握所带班级（可跨多个班）的打卡完成情况 | 查看实时数据 → 识别异常 → 复核心得 → 导出数据 |
| **管理员** | 统计全校打卡数据，生成思政教育工作报告 | 发布任务 → 管理组织 → 查看报表 → 导出报告 |

> **关键假设**：学生主要因学校/课程要求而打卡，产品通过降低操作摩擦和轻量激励提升完成率与体验。

## 3. 核心场景

### 场景一：学生完成每日打卡

1. 打开 App，首页显示每日名言/正能量语录。
2. 查看当日发布的思政学习任务。
3. 阅读任务内容（文章、视频、通知等）。
4. 点击「定位签到」，上传当前位置信息（V1 仅记录，不做范围校验）。
5. 撰写并提交学习心得。
6. 系统通过 AI 初审；若通过，打卡完成并获得积分；若未通过，进入辅导员人工复核。
7. 查看个人打卡日历、连续打卡天数、积分等级和班级排行榜。

### 场景二：辅导员查看班级数据

1. 登录后查看所带班级的实时打卡数据（允许数分钟延迟）。
2. 查看班级打卡率、未打卡学生名单、待复核心得列表。
3. 对 AI 初审未通过的心得进行人工复核。
4. 导出班级打卡数据用于督促或上报。

### 场景三：管理员生成全校报告

1. 登录后按学院、班级、时间段筛选数据。
2. 查看全校打卡率、各学院/班级排名、趋势变化等可视化图表。
3. 一键导出报告（PDF/Excel），用于向上级汇报。

## 4. 核心功能边界

### 4.1 学生端

- 每日打开 App 显示名言/正能量语录
- 查看当日/历史任务
- 定位签到（V1 仅上传位置，不做围栏校验）
- 提交学习心得/体会
- 查看个人打卡日历、连续打卡天数与勋章
- 查看积分与等级
- 查看班级排行榜（班级整体数据，不展示个人排名）
- 接收打卡提醒（可选）

### 4.2 辅导员端

- 管理所带班级
- 查看班级实时打卡数据（允许数分钟延迟）
- 查看未打卡学生名单
- 人工复核 AI 初审未通过的心得
- 导出班级数据（Excel）

### 4.3 管理员端

- 发布/管理思政学习任务
- 管理学院、班级、辅导员、学生账号
- 配置每日名言/语录库
- 查看全校统计数据与可视化报表，包括：
  - 整体打卡率（全校/学院/班级）
  - 未打卡/异常名单
  - 班级/学院排名
  - 日/周/月趋势
  - 心得提交与审核情况
- 导出全校报告（Excel/PDF）

### 4.4 积分与等级机制

- 每次打卡获得基础积分。
- 连续打卡可获得加成积分（如连续 7 天积分翻倍）。
- 积分累计提升等级（如青铜 → 白银 → 黄金）。
- 积分与等级在个人页展示，增强成就感。
- V1 不包含积分兑换奖品/商城闭环。

### 4.5 审核机制

- **AI 初审**：自动检测心得是否敷衍（字数过少、复制粘贴、无意义内容）以及是否包含敏感词。
- **人工复核**：AI 初审未通过的内容自动推送给对应辅导员；辅导员可判定「通过」「不通过」或「要求修改」。

## 5. 非目标 / 超出范围

为控制实习作业的 scope，V1 不实现：

- 积分兑换奖品/商城运营闭环（详见 `addendum.md`）
- 地理位置围栏校验（V1 仅记录位置）
- 个人排行榜/同学互评等社交功能
- 即时通讯/社交动态（班级群聊、评论、点赞等）
- 多学校/SaaS 化（单一学校部署）
- 复杂离线同步
- 深度 AI 分析（情感分析、学习效果评估等）

## 6. 成功标准与度量

| 维度 | 指标 | 验证方式 |
|------|------|---------|
| 功能完整 | 学生打卡、辅导员查看、管理员统计三条核心链路 100% 可跑通 | 功能测试 |
| 界面美观 | 视觉风格统一，无明显 UI 错位或风格冲突；打卡日历、班级榜等可视化组件有设计感 | 设计走查 |
| 用户友好 | 学生可在 3 步内完成一次打卡 | 任务测试 |
| 用户友好 | 辅导员可在 2 步内查看所带班级当日打卡率 | 任务测试 |
| 数据准确 | 报告中的打卡率、未打卡名单与原始数据一致 | 数据核对 |
| 激励感知 | 学生能清晰看到自己的积分、等级、连续打卡和班级排名 | 功能测试 |

## 7. 风险与关键假设

### 7.1 关键假设

- 思政学习任务由管理员或辅导员统一发布，学生按任务打卡。
- 辅导员愿意并能够处理 AI 初审未通过的复核工作。
- 学校允许采集并存储学生位置信息与心得内容用于思政教育管理。
- 管理员报告的对象为上级主管部门或校级领导。
- 轻量激励（积分、日历、班级榜）能提升学生完成率，但不会显著增加开发复杂度。

### 7.2 主要风险

- **AI 审核准确率**：AI 可能误判敷衍内容或漏过敏感信息，需要人工兜底。
- **学生隐私与数据安全**：位置信息和学习心得属于敏感数据，需要妥善存储和访问控制。
- **辅导员工作负担**：若 AI 初审阈值过严，可能导致大量内容进入人工复核。
- **学生接受度**：若打卡流程繁琐或体验不佳，可能出现敷衍打卡、代打卡等行为。
- **积分公平性**：连续打卡加成可能让后期加入的学生难以追赶，需考虑是否设置合理上限。
- **合规要求**：需符合学校及教育行业对数据与内容的合规要求。
