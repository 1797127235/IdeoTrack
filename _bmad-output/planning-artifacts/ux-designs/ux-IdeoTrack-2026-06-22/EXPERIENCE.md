# 思政打卡 App — EXPERIENCE.md

> 视觉参考：`DESIGN.md`
> 需求来源：`prd.md`
> 状态：ready-for-review

## Foundation

### Form-Factor

- **主要平台**：Android 手机（以 375–414px 宽度为设计基准）
- **跨平台**：React Native 代码同时编译到 iOS，视觉和交互保持 95% 一致；仅平台级行为（状态栏、返回手势、底部安全区）按平台适配。
- **不支持的形态**：平板、桌面、Web（V1 专注手机端）。
- **关键界面原型**：见 `mockups/index.html`，包含学生首页、任务详情、打卡成功、日历、辅导员看板等 5 个高保真 HTML 原型。

### UI System

- 不绑定特定 UI 库（如 React Native Paper/NativeBase）。
- 自定义组件基于 `DESIGN.md` 的 tokens 构建，保持风格统一。
- 使用 React Native 基础组件 + 少量社区库：
  - 导航：React Navigation（底部 Tab + 栈导航）
  - 图表：react-native-chart-kit 或 victory-native
  - 日期选择：@react-native-community/datetimepicker
  - 图标：lucide-react-native

## Information Architecture

### 学生端 Surface

| 页面 | 入口 | 用途 |
|------|------|------|
| 首页 | 底部 Tab「首页」 | 每日名言 + 今日任务 + 快速打卡 + 连续打卡/积分 |
| 任务详情 | 首页任务卡片 | 阅读任务内容 + 定位签到 + 提交心得 |
| 打卡成功 | 提交心得后 | 反馈积分、连续天数、勋章获得 |
| 日历 | 底部 Tab「日历」 | 查看历史打卡记录和连续打卡情况 |
| 排行榜 | 底部 Tab「排行」 | 查看班级打卡率排名 |
| 我的 | 底部 Tab「我的」 | 个人资料、积分等级、设置、通知 |

### 辅导员端 Surface（微信小程序）

辅导员端与学生端共用同一个微信小程序，登录后按角色进入辅导员视图。

| 页面 | 入口 | 用途 |
|------|------|------|
| 看板 | 登录后首页 | 所带班级打卡率概览 |
| 班级详情 | 看板班级卡片 | 未打卡名单、已打卡名单、班级统计 |
| 待复核 | 底部 Tab「复核」 | AI 初审未通过的心得列表 |
| 复核详情 | 待复核列表项 | 查看心得并判定通过/不通过/要求修改 |
| 数据导出 | 班级详情页入口 | 生成导出链接并复制到浏览器下载 |
| 我的 | 底部 Tab「我的」 | 个人资料、设置 |

### 管理员端 Surface

| 页面 | 入口 | 用途 |
|------|------|------|
| 数据概览 | 登录后首页 | 全校/学院/班级打卡率 KPI |
| 统计报表 | 底部 Tab「报表」 | 多维度图表 + 数据筛选 |
| 任务管理 | 底部 Tab「任务」 | 发布/编辑/下架任务 |
| 组织管理 | 底部 Tab「管理」 | 学院/班级/用户/名言库管理 |
| 我的 | 底部 Tab「我的」 | 个人资料、设置 |

### 导航结构

- **学生（小程序）**：底部 4 Tab（首页 / 日历 / 排行 / 我的）
- **辅导员（小程序）**：底部 3 Tab（看板 / 复核 / 我的）
- **管理员（Expo App / V2 Web）**：底部 4 Tab（概览 / 报表 / 任务 / 管理）
- App 端所有页面通过栈导航（Stack Navigator）进入二级详情；小程序端通过页面跳转（`wx.navigateTo`）进入二级详情。

## Voice and Tone

### 整体语气

**友好、鼓励、清晰、不命令**。学生完成的是学校任务，产品不制造焦虑，而是用正向反馈引导行为。

### 微文案规范

| 场景 | Do | Don't |
|------|-----|------|
| 打卡成功 | 「今日打卡完成！连续 5 天，继续加油！」 | 「打卡成功」 |
| 未打卡提醒 | 「今天还没有打卡哦，3 分钟就能完成～」 | 「你还未打卡，请尽快完成」 |
| AI 初审未通过 | 「心得可以再具体一点，说说你的真实感受吧」 | 「内容敷衍，重新提交」 |
| 空状态 | 「还没有打卡记录，从今天的任务开始吧」 | 「暂无数据」 |
| 辅导员复核 | 「这条心得需要你的判断」 | 「待审核」 |

### 文案禁忌

- 不使用「必须」「立刻」「否则」等强制词汇。
- 不使用过度夸张的 emoji 或感叹号。
- 不对学生个人进行负面评价（如「你太懒了」）。

## Component Patterns

### 任务卡片

- 位置：学生首页首屏
- 结构：
  1. 标签：「今日任务」
  2. 标题：任务名称
  3. 摘要：任务内容前 2 行
  4. 元信息：截止时间 + 预计时长
  5. 状态：未开始 / 进行中 / 已完成 / 审核中
- 行为：点击整卡进入任务详情。

### 打卡按钮

- 位置：任务详情页底部固定区域
- 状态：
  - 默认：「立即打卡 +10 积分」
  - 加载中：「正在提交...」+ spinner
  - 已打卡：「今日已打卡」禁用态
- 行为：点击后进入定位签到 + 心得输入流程。

### 心得输入框

- 位置：任务详情页签到完成后
- 结构：多行文本输入 + 字数统计
- 提示文案：「写下你的学习心得（最少 10 字）」
- 错误态：字数不足时边框变红，提示「再多写一点吧～」

### 数据概览卡片（辅导员/管理员）

- 结构：大数字 + 标签 + 环比箭头（可选）
- 示例：「92% 今日打卡率」「120 应打卡人数」「110 已打卡人数」
- 行为：点击下钻到详情列表或图表。

### 班级排行榜项

- 结构：排名序号 + 班级名称 + 打卡率进度条
- 前三名使用金银铜徽章或色块强调。
- 不展示个人姓名或学号。

## State Patterns

### 首页加载

- 有缓存：先展示缓存内容，后台静默刷新。
- 无缓存：显示骨架屏（任务卡片、日历、积分区）。
- 加载失败：显示「网络不太稳，点击重试」，保留缓存数据。

### 任务状态

| 状态 | 视觉表现 | 操作 |
|------|---------|------|
| 未开始 | 卡片正常，CTA 按钮可用 | 点击去打卡 |
| 进行中 | 同上 | 同上 |
| 审核中 | 卡片显示「审核中」标签，按钮禁用 | 查看详情 |
| 已完成 | 卡片显示「已完成」绿色标签，按钮变为「今日已打卡」 | 查看心得 |
| 已逾期 | 卡片变灰，显示「已逾期」标签 | 不可打卡（V1 不支持补卡） |

### 打卡成功反馈

- 页面跳转：从任务详情 → 打卡成功页
- 成功页内容：
  - 大号成功图标（绿色对勾动画）
  - 文案：「今日打卡完成！」
  - 积分 +10、连续天数、等级进度
  - 若获得勋章，展示勋章动画
  - 底部按钮：「返回首页」「查看日历」

### 空状态

- 无任务：「今天没有发布任务，去日历看看历史记录吧」
- 无打卡记录：「还没有打卡记录，从今天的任务开始吧」
- 无待复核：「暂无需要复核的心得，AI 初审都通过了～」
- 无数据权限：「你没有权限查看这个班级的数据」

### 错误状态

- 定位失败：提示「请开启定位权限以完成签到」，提供「去设置」按钮。
- 网络提交失败：保留用户输入，提示「提交失败，点击重试」。
- AI 初审失败：显示「系统繁忙，已转交辅导员复核」。

## Interaction Primitives

### 手势

- **点击**：主要交互方式。
- **滑动**：
  - 首页名言：左右滑动查看近 7 日名言。
  - 列表：标准滚动，不使用侧滑删除（避免误操作）。
- **下拉刷新**：辅导员看板、管理员报表、排行榜支持下拉刷新。
- **上拉加载**：历史任务列表、通知列表支持分页加载。

### 反馈

- **按钮按下**：scale(0.98)，时长 100ms。
- **打卡成功**：绿色对勾动画 + 轻微震动（haptic feedback）。
- **获得勋章**：徽章弹出 +  confetti 效果（可选，V1 可用简单缩放动画）。
- **提交加载**：按钮显示 spinner，禁用重复提交。
- **Toast**：底部弹出，2 秒后自动消失，用于轻量提示（如「提醒已发送」）。

### 转场

- 页面 push：从右向左滑入，250ms。
- 页面 pop：从左向右滑出，250ms。
- Modal 底部 Sheet：从下向上滑入，300ms。

## Accessibility Floor

### 触控

- 所有可点击元素热区 ≥ 44×44pt（iOS）/ 48dp（Android）。
- 按钮之间保留足够间距，避免误触。

### 视觉

- 正文对比度 ≥ 4.5:1（已通过 `DESIGN.md` 色值保证）。
- 不使用颜色作为唯一状态标识（如错误状态同时有文字和图标）。
- 图标按钮必须有 `accessibilityLabel`。

### 屏幕阅读器

- 每个页面标题通过 `AccessibilityInfo` 或导航标题声明。
- 图片/图标有描述性标签。
- 表单输入关联 label。
- 动态内容更新（如 Toast）通过屏幕阅读器播报。

### 动画

- 尊重系统 `prefers-reduced-motion` 设置。
- 核心功能不依赖动画完成（动画是增强，不是必需）。

## Responsive & Platform

### 屏幕适配

- 设计基准：375px 宽（iPhone SE / 小型 Android）。
- 适配范围：320px–428px 手机宽度。
- 大屏手机（>414px）：内容居中，最大宽度 428px；卡片和列表适当增宽，不拉伸变形。

### 平台差异

| 元素 | Android | iOS |
|------|---------|-----|
| 状态栏 | 深色图标，背景随 Header 渐变 | 系统默认，背景随 Header 渐变 |
| 返回手势 | 边缘侧滑返回 | 边缘侧滑返回 |
| 底部导航 | 符合 Material 高度 | 符合 iOS Tab Bar 规范 |
| 字体 | 使用 Noto Sans SC | 使用 Noto Sans SC |
| 日期选择器 | @react-native-community/datetimepicker | 同上 |

### 安全区

- 使用 `SafeAreaView` 处理刘海屏/灵动岛/底部手势条。
- 底部 Tab 栏高度：Android 56dp，iOS 49pt + bottom inset。

## Key Flows

### Flow 1 — 小林早晨完成打卡

1. 小林打开 App，首页加载今日名言和任务卡片。
2. 看到「学习《新时代青年使命担当》」任务卡片，点击进入详情。
3. 阅读任务内容，点击底部「立即打卡」按钮。
4. App 请求定位权限，自动记录当前位置。
5. 进入心得输入页，写下 3 句话学习体会，字数满足要求。
6. 点击「提交心得」，显示 loading。
7. AI 初审通过，跳转到打卡成功页。
8. **Climax：** 页面显示「今日打卡完成！」、积分 +10、连续 5 天、勋章获得动画。
9. 小林点击「返回首页」，首页任务卡片状态变为「已完成」。

**失败路径：** 若 AI 初审判定敷衍，停留在输入页，提示「心得可以再具体一点」，显示字数不足原因。

### Flow 2 — 王老师查看班级并提醒未打卡学生

1. 王老师登录后进入辅导员看板，看到 3 个班级的今日打卡率。
2. 点击打卡率最低的班级，进入班级详情。
3. 查看「未打卡」标签页，看到 10 名学生名单。
4. 勾选全部 10 人，点击「一键提醒」。
5. **Climax：** 系统提示「已发送提醒给 10 名学生」，名单中出现「已提醒」标记。
6. 切换到「待复核」标签，逐条处理 AI 初审未通过的心得。

**失败路径：** 若某学生连续 3 天未打卡，名单上该学生所在行标红，提示「重点关注」。

### Flow 3 — 李主任生成月度报告

1. 李主任登录后进入管理员数据概览页。
2. 点击底部 Tab「报表」，进入统计报表页。
3. 选择时间范围为「本月」，查看全校打卡率趋势图和各学院排名。
4. 点击某个学院下钻，查看该学院各班级完成情况。
5. 点击「导出报告」，选择 PDF 格式。
6. **Climax：** 系统生成报告，提示「报告已生成，点击下载」。
7. 李主任下载 PDF，用于向上级汇报。

**失败路径：** 若某天打卡率较 7 天均值下降 ≥20 个百分点，趋势图上该点标注「建议核查」。

## Inspiration & Anti-patterns

### 借鉴

- **Duolingo**：连续打卡和成就系统的设计语言（但降低游戏化强度）。
- **GitHub Contribution Graph**：打卡日历的绿色格子视觉反馈。
- **学习通/钉钉**：任务列表和信息层级清晰，适合教育场景。

### 避免

- **过度游戏化**：不使用强烈音效、震动轰炸、强制分享。
- **暗黑模式压迫感**：V1 浅色模式为主，避免深色背景上的高饱和色彩。
- **复杂导航**：不使用抽屉导航、汉堡菜单，底部 Tab 足够覆盖 V1 功能。
- **个人排行榜公开竞争**：保护学生隐私，只展示班级整体排名。
